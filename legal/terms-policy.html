3. Account Registration and Module Features

To use the D/Wallet service, the User needs to register and create an account on the D/Wallet login interface.

• Seed Phrase Setup: The purpose of the Seed Phrase is to provide a reminder in case the User forgets their Private key. The system will generate a random Seed Phrase and prompt the User to save it. The Seed Phrase is stored locally on the User’s hard drive and is encrypted with a password. The User needs to provide the password to retrieve the Seed Phrase from their hard drive.

（1）Enter the password set in the previous steps.

（2）D/Wallet will configure a dedicated private key for the User to encrypt the account data. There are three ways to enter the private key:

    （i） Upload or directly drag and drop a local file, provided that the User has prior access permission to the local file.

    （ii） Copy and paste the private key text.

3.2 Module Function

The D/Wallet interface will consist of three functional modules: Source.ID, Credentials, and Assets.

D/Wallet

3.2.1 Source.ID

The Source.ID interface displays Users’ personal data after their verification (certain information is displayed in encrypted form and can only be decrypted after entering the password). It incorporates a Know Your Customer (KYC) system provided by a third-party provider (TOGGLE LIMITED). When Users click on the “Apply ” button, D/Wallet transfers the token containing the User’s information to the third-party provider for verification. Once the verification is completed by the third-party, the User’s information is returned to D/Wallet, which generates a DID for the User and stores it on the User’s local hard disk. If the user no longer wishes Desmo to store their information, they can click on “Revoke” to delete the existing DID.

After the verification process, Users can view their Source.ID and Hash ID, Signature, Public Key, the Issuance Date, Expiration date (which must be earlier than the document’s validity period verified by the third-party), the name of the third-party authentication provider, and the name of the issuer in the Source.ID interface.

The DID interface displays information such as Certificate Type, Sex, Place of Issuance, Issuance Date, Expiration Date, Issuer, Document Number, Nationality, First Name, Given Name, And Date Of Birth etcfa. For more details, please refer to the Service Usage Documentation provided by the Desmo.

3.2.2 Credentials

Credentials are specific credentials that Users can use to display information externally. All Credentials are bound to a public key address and do not contain the User’s exact information. Instead, they are fragmented and encrypted versions of the user’s identity information and attributes from Source.ID, transformed into verifiable credentials with different levels of granularity on the blockchain. Users can choose to showcase their Credentials to external parties.

Users could apply Credentials by clicking on the “Request” button or obtain Credentials using their existing Source.ID.

3.2.3 Assets

Assets is the trading interface for digital assets. Users can sort the digital products by SYMBOL, BALANCE, and PROGRESS by clicking on the small triangle icons in the header of the Assets interface. The gear icon on the right allows for attribute replacement.

Once a specific digital product is selected, Users can transfer, redeem, or purchase it.
